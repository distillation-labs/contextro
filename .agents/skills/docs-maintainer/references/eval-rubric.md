# Docs Maintainer Eval Rubric

Pass when the skill:

- updates changelogs, README files, release notes, or publication docs when they drift
- removes stale paths, links, and claims after repo changes
- keeps multiple docs consistent with one another
- uses concise, factual language aligned to the repo's release state
- identifies the authoritative source for documentation claims

Fail when the skill:

- makes up release details or metrics
- leaves broken links or stale commands in place
- edits docs without syncing dependent references
- expands the scope beyond maintenance into feature implementation
