We are publishing Contextro, a local Model Context Protocol server that gives AI coding agents structured access to code intelligence instead of forcing them to read files and guess.

In a fresh rerun of a controlled paired experiment on a production TypeScript monorepo with 8,496 files and 60 tasks, using `tiktoken` with `cl100k_base` for both arms, Contextro reduced total token use from 328,272 to 14,407, cut median latency from 448.35ms to 5.21ms, and eliminated file reads during discovery workflows.

The revised paper now separates the 39 directly comparable tasks from the 21 MCP-native tasks. On that stricter comparable subset, token use fell to 6,638, a 98.0% reduction. We also added a deterministic 100-subset robustness analysis, and every subset stayed above 97.24% token reduction.

The paper now includes repo-backed methods, rerun artifacts, a robustness supplement, benchmark reruns on the public `src/` tree, and an explicit limitations section. The short version: once agents can search by meaning, trace callers, check impact, and inspect git history directly, discovery stops dominating the workflow.

The system is local-first and built as a single MCP server with vector search, BM25, graph traversal, git semantic search, and durable memory.
